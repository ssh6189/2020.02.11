OSI 참조 모델 7계층

계층 7개는 서로 독립적이므로 어느 한계층의 변경이 다른 계층에는 영향을 미치지 않는다.

기능에 필요한 몇개의 계층만 표준화하면, 정상적으로 통신할 수 있다.

만약 127.0.0.1로 통신하면, 전송계층까지 왔다가, 다시 응용계층으로 간다.

네트워크를 사용하는 순간부터, 아래로 내려간다. 밑의 네트워크 계층, 데이터 링크 계층, 물리계층 이 3개는
랜카드가 없어도 가능하다.

(*인터넷 선과 랜은 다르다!)



전송계층 : 프로토콜(TCP, UDP)와 관련된 계층으로 오류복구와 흐름제어 등을 담당하며, 두 시스템 간에 신뢰성 있는
데이터를 전송한다.

또한, 네트워크 계층에서 온 데이터를 세션 계층의 어느 애플리케이션에 보낼 것인지 판독하고 네트워크 계층으로 전송할 경로를 선택한다.

OSI 참조모델 7계층 중 전송 계층은 네번째 계층으로 시스템 종단 간에 투명한 데이터를 양방향으로 전송하는 계층이다.

네트워크 계층에서 전송한 데이터와 실제 운영체제의 프로그램이 연결되는 통신 경로라고 할 수 ㅣㅇㅆ다.



(네트워크로 데이터를 보내면, 중간에 확실히 전송되는지 확실히 알 수 없다.)

송신측 -> (오류복구/흐름제어) -> 수신측

TCP는 수신측이 받고, 받았다는 응답신호를 보내준다.

UDP는 수신측이 받아도, 받았다는 신호를 안보내준다. 그대신 고속이다.                                                                              

요새는 네트워크가 많이 빨라져서, UDP 사용이 많이 줄었다.

신뢰성이 더 중요하기 때문이다.



OSI 참조모델 7계층

응용계층

응용계층과 사용자(사람 또는 소프트웨어), 표현 계층간의 관계를보여준다.



전송계층
인터넷 모델의 전송 계층에는 TCP와 UDP 프로토콜이 2개 있다.

TCP는 송신지에서 수신지까지 문자 스트림을 전송하는데, 두 응용 계층이 서로 대화하는것을 허용하는 신뢰성
있는 프로토콜이다.

(TCP의 성능은 OSI참조 모델의 전송계층보다 뛰어나다.)

UDP : ( 사용자 데이터 그램 프로토콜)은 OSI 참조 모델에서 정의하는 전송 계층의 일부 역할을 무시하는 단순한 전송 프로토콜이다.

UDP는 TCP에 비해 신뢰성이 낮다.
즉, Hello -> Helo 이런식으로 전송해준다.

TCP를 사용한 전송계층의 예를 살펴보자, 송신 측에서 데이터를 보내면, TCP의 포트번호 80버늘 이용하여
수신측으로 데이터를 안전하게 전송한다.
(포트번호는 16비트)
전송계층에서는 송신지에서 수신지까지 메세지 전송기능을 제공한다.



응용계층
인터넷 모델의 응용계층에 포함되어 있는 프로토콜 7개와 프로그램은 원격으로 컴퓨터 자원에 접속하는데 사용한다.



응용프로그램들로 제공되는 서비스는 표현 계층과 세션계층에서 정의하고 있다.



TCP/IP의 주소

인터넷에 연결된 모든 컴퓨터에는 고유의 주소가 부여되는데, 이를 'IP주소'라고 한다.

현재 사용하는 IP 주소체계는 IP Ver.4


포트번호 : 포트는 TCP가 상위 계층으로 데이터를 전달하거나, 상위계층에서 TCP로 데이터를 전달할 때
상호간에 사용하는 데이터의 이동 통로를 말한다.

통신할 때 여러 웹사이트에서 파일을 동시에 다운 받을 수 있다.
